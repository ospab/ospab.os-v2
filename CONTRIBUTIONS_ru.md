# Участие в разработке AETERNA / ospab.os

Благодарим за интерес к проекту. Этот документ описывает среду сборки,
соглашения по коду и рабочий процесс для контрибьюторов.

---

## Начало работы

### Требования

| Инструмент | Назначение | Установка |
|---|---|---|
| Rust nightly | Компиляция ядра и юзерленда | `rustup default nightly` |
| `xorriso` | Генерация гибридного ISO | `apt install xorriso` |
| `mtools` | Работа с образами FAT32 | `apt install mtools` |
| LLVM toolchain | `llvm-objcopy` для strip, `clang` для Си | `apt install llvm clang` |
| QEMU | Тестирование | `apt install qemu-system-x86` |

Ядро собирается под кастомный Rust-таргет `x86_64-ospab`, определённый в `x86_64-ospab.json`.
`rustup target add` не требуется — cargo подхватывает JSON-таргет автоматически
через флаг `-Z build-std` в `.cargo/config.toml`.

### Клонирование и сборка

```bash
git clone https://github.com/ospab/aeterna.git
cd aeterna
bash build.sh
```

### Запуск

```bash
# Минимальная загрузка QEMU
qemu-system-x86_64 -cdrom isos/ospab-os-v2-103.iso -m 256M -serial stdio

# С поддержкой сети RTL8139
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-103.iso \
  -m 256M -serial stdio \
  -device rtl8139,netdev=net0 \
  -netdev user,id=net0

# С дисковым хранилищем
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-103.iso \
  -m 256M -serial stdio \
  -drive file=disk.img,format=raw,if=ide
```

## Структура кодовой базы

| Путь | Описание |
|---|---|
| `src/main.rs` | Точка входа — 5-фазная инициализация |
| `src/lib.rs` | Корень крейта — все объявления модулей |
| `src/terminal.rs` | Интерактивный терминал, диспетчер команд |
| `src/installer.rs` | UEFI-установщик (GPT + FAT32 + LFN) |
| `src/doom.rs` | FFI-слой Rust для движка DOOM |
| `src/fs/` | VFS + RamFS + отложенная синхронизация |
| `src/net/` | RTL8139, e1000, Ethernet, ARP, IPv4, ICMP, UDP, SNTP |
| `src/drivers/` | ATA PIO, AHCI SATA, NVMe, AC97, ES1371 |
| `arch/x86_64/` | GDT, IDT, PIC, клавиатура, фреймбуфер, serial, panic |
| `core/` | Планировщик, IPC, диспетчер сисколов, модель capability |
| `mm/` | Физический аллокатор, куча, VMM |
| `drivers/` | Перечисление PCI, трейты хранилища/видео/VirtIO |
| `userland/grape/` | Текстовый редактор |
| `userland/tomato/` | Пакетный менеджер |
| `userland/plum/` | Оболочка |
| `userland/seed/` | Система инициализации |
| `doom_engine/` | Исходники doomgeneric на Си + прослойка libc |

## Соглашения по коду

### Rust

- `#![no_std]` везде — стандартная библиотека не используется. Используйте `extern crate alloc` для `Vec`, `String`, `BTreeMap`.
- Все публичные функции и типы требуют doc-комментариев (`///`).
- Блоки `unsafe` требуют комментария `// SAFETY:` с объяснением соблюдаемого инварианта.
- Используйте `crate::` для внутренних путей, не `ospab_os::`.
- Функции доступа к железу должны быть `unsafe` и вызываться из единственного места инициализации.

### Именование

| Элемент | Стиль | Пример |
|---|---|---|
| Модули, функции, переменные | `snake_case` | `init_driver`, `send_packet` |
| Типы, трейты, перечисления | `PascalCase` | `PacketBuffer`, `ServiceState` |
| Константы | `SCREAMING_SNAKE_CASE` | `MAX_PACKET_SIZE` |
| Файлы | `snake_case.rs` | `rtl8139.rs`, `ramfs.rs` |

### Организация файлов

- Каждая подсистема — отдельный каталог с `mod.rs` в качестве точки входа.
- Определения трейтов выносятся в отдельный `trait.rs` в том же каталоге.
- Инструменты юзерленда живут в `userland/<tool>/src/lib.rs`.
- Архитектурно-специфичный код — в `arch/x86_64/`.

### Логирование

- Все сообщения подсистем используют префикс: `[NET]`, `[FS]`, `[ATA]`, `[DOOM]` и т.д.
- Сообщения завершаются `\r\n` для совместимости с serial.
- Сообщения фаз загрузки используют `klog::boot()` / `klog::err()` — для кольцевого буфера `dmesg`.
- Отладочные трассировки через `serial::write_str()` необходимо удалять перед слиянием.

## Области для участия

### Высокий приоритет

- **Изоляция процессов** — загрузчик ELF, Ring 3, ABI сисколов, `fork`/`exec`/`wait`
- **TCP/IP** — автомат состояний SYN/ACK, таблица соединений, таймер повторной передачи
- **Капабилити** — таблица capability на процесс, интеграция в диспетчер сисколов

### Средний приоритет

- **Вытесняющий планировщик** — TCB, переключение контекста по IRQ 0, классы приоритетов
- **USB-стек** — xHCI, HID-клавиатура, Mass Storage / BOT
- **Монтирование ext2/FAT32** — чтение существующих ФС с диска в VFS

### Хорошие задачи для начала

- Добавить новую команду в терминал (см. блок `match cmd_name` в `src/terminal.rs`)
- Расширить темы `tutor` техническим содержанием
- Добавить пакеты в встроенный репозиторий `tomato`
- Написать тесты для VFS (без железа — RamFS это чистый Rust)
- Исправить предупреждения: `cargo clippy`

## Рабочий процесс

### 1. Форк и ветка

```bash
git checkout -b feat/my-feature
```

### 2. Разработка

- Тестируйте в QEMU после каждого значимого изменения.
- Убедитесь, что `bash build.sh` завершается без ошибок и предупреждений.
- Проверяйте вывод `dmesg` в запущенной ОС.

### 3. Сообщения коммитов

Используйте формат [Conventional Commits](https://www.conventionalcommits.org/):

```
feat(net): реализовать автомат состояний SYN/ACK TCP
fix(vfs): обработать параллельный readdir во время отложенной синхронизации
refactor(terminal): вынести диспетчер команд в отдельный модуль
docs: обновить ROADMAP с прогрессом Фазы 9
chore(build): добавить шаг strip с llvm-objcopy
```

Скоупы: `kernel`, `net`, `vfs`, `fs`, `mm`, `arch`, `installer`, `doom`, `terminal`,
`plum`, `grape`, `tomato`, `seed`, `drivers`, `build`, `docs`.

### 4. Пул-реквест

Укажите:
- Чёткое описание изменения и его цели.
- Шаги для воспроизведения/тестирования.
- Вывод serial (`-serial stdio`) с успешным прохождением затронутых фаз загрузки.
- Скриншот, если изменяется вывод фреймбуфера.

## Тестирование

Поскольку AETERNA — это bare-metal ядро, всё тестирование выполняется через QEMU:

```bash
bash build.sh
qemu-system-x86_64 -cdrom isos/ospab-os-v2-103.iso -m 256M -serial stdio
```

**Минимальный чеклист перед отправкой PR:**

- [ ] Все 5 фаз загрузки завершились с `[OK]`
- [ ] Приглашение оболочки появилось (`root@ospab:~# `)
- [ ] `version` показывает корректную версию ядра
- [ ] `ls /` возвращает стандартное дерево каталогов
- [ ] `ping 10.0.2.2` возвращает RTT в микросекундах (при изменении сети)
- [ ] `grape /tmp/test.txt` открывается, принимает ввод, сохраняет, корректно выходит (при изменении редактора)
- [ ] `seed status` показывает все 9 сервисов (при изменении init)
- [ ] `bash build.sh` — ноль предупреждений
