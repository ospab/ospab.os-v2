# Участие в разработке ospab.os

Спасибо за интерес к проекту **ospab.os v2.1.0**! В этом документе описаны правила и рабочий процесс для участия в разработке.

## Начало работы

### Предварительные требования
- **Rust nightly** (`rustup default nightly`)
- **xorriso** и **mtools** для генерации ISO
- **QEMU** для тестирования (`qemu-system-x86_64`)
- **WSL** или Linux-окружение для сборки

### Сборка
```bash
git clone https://github.com/user/ospab.os-v2.git
cd ospab.os-v2
bash build.sh
```

### Запуск
```bash
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-19.iso \
  -m 256M \
  -serial stdio \
  -device rtl8139,netdev=net0 \
  -netdev user,id=net0
```

## Архитектура проекта

Кодовая база организована по следующим ключевым областям:

| Область | Путь | Описание |
|---------|------|----------|
| Ядро | `src/main.rs`, `src/lib.rs` | Последовательность загрузки, объявления модулей |
| HAL | `arch/x86_64/` | GDT, IDT, PIC, клавиатура, фреймбуфер |
| Память | `mm/` | Физический аллокатор, куча, VMM |
| Файловая система | `src/fs/` | Реализация VFS + RamFS |
| Сеть | `src/net/` | RTL8139, Ethernet, ARP, IPv4, ICMP, UDP |
| Терминал | `src/terminal.rs` | Диспетчер команд, 28+ команд |
| Юзерленд | `userland/` | grape, tomato, plum, seed |
| Драйверы | `src/drivers/`, `drivers/` | ATA, AHCI, PCI |

## Области для вклада

### Высокий приоритет
- **Изоляция процессов** — ELF-загрузчик, исполнение в Ring 3, IPC
- **Исправление RTL8139 RX** — приём возвращает CBR=0, требуется исследование DMA-выравнивания
- **Стек TCP/IP** — сейчас реализован только UDP
- **Файловая система ext2/FAT** — постоянное хранилище помимо RamFS

### Средний приоритет
- **Дополнительные драйверы NIC** — e1000, VirtIO-net
- **Стек USB** — UHCI/EHCI/xHCI
- **Звуковой драйвер** — AC97 или Intel HDA
- **Графика** — двойная буферизация фреймбуфера, прототип оконного менеджера

### Хорошие первые задачи
- Добавление новых команд в терминал
- Расширение тем `tutor` дополнительным содержанием
- Добавление новых пакетов в репозиторий `tomato`
- Улучшение скриптинга `plum` (циклы, условия)
- Написание тестов для операций VFS

## Стиль кода

### Правила Rust
- Используем `#![no_std]` — без стандартной библиотеки
- Все аллокации через крейт `alloc` (`Vec`, `String`, `BTreeMap`)
- Используем `crate::` для внутренних ссылок (не `ospab_os::`)
- Unsafe-код допустим для доступа к железу, но должен быть документирован
- Все публичные функции должны иметь doc-комментарии

### Соглашения об именовании
- **Модули**: `snake_case` (например, `src/net/rtl8139.rs`)
- **Типы**: `PascalCase` (например, `PacketBuffer`, `ServiceStatus`)
- **Функции**: `snake_case` (например, `init_driver`, `send_packet`)
- **Константы**: `SCREAMING_SNAKE_CASE` (например, `MAX_PACKET_SIZE`)

### Организация файлов
- Каждая подсистема имеет свой каталог с `mod.rs`
- Инструменты юзерленда находятся в `userland/<tool>/src/lib.rs`
- Модули юзерленда подключаются через `#[path]` в `src/lib.rs`
- Определения трейтов — в отдельных файлах `trait.rs`

## Рабочий процесс

### 1. Форк и ветка
```bash
git checkout -b feature/my-feature
```

### 2. Внесение изменений
- Пишите код согласно стилевому руководству
- Тестируйте в QEMU
- Убедитесь, что `bash build.sh` завершается без ошибок

### 3. Коммит
Используйте ясные описательные сообщения коммитов:
```
feat(net): добавить TCP SYN/ACK хендшейк
fix(vfs): обработка создания вложенных каталогов
refactor(terminal): вынести диспетчер команд в отдельный модуль
docs: обновить README с новыми командами
```

### 4. Отправка
Откройте pull request с:
- Описанием изменений
- Шагами тестирования
- Скриншотами/логами при необходимости

## Тестирование

Так как ospab.os — bare-metal ОС, тестирование проводится через QEMU:

```bash
# Сборка и запуск
bash build.sh
qemu-system-x86_64 -cdrom isos/ospab-os-v2-19.iso -m 256M -serial stdio

# С сетью
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-19.iso \
  -m 256M \
  -serial stdio \
  -device rtl8139,netdev=net0 \
  -netdev user,id=net0

# С хранилищем
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-19.iso \
  -m 256M \
  -serial stdio \
  -drive file=disk.img,format=raw,if=ide
```

### Что тестировать
- Загрузка завершает все 5 фаз
- Терминал отвечает на команды
- `grape <file>` — редактор открывается, можно печатать, сохранять, выходить
- `tomato -S base` — пакет устанавливается
- `seed status` — показывает сервисы
- `export VAR=test && echo $VAR` — переменные оболочки работают
