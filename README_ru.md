# ospab.os v2 — Микроядерная операционная система AETERNA

> **Версия 2.1.0** | **Ядро: AETERNA** | **Архитектура: x86_64** | **Язык: Rust (no_std)**

## О проекте

**ospab.os** — микроядерная операционная система, полностью написанная на Rust. Ядро — **AETERNA** — обеспечивает детерминированное планирование, безопасность на основе capability и вычислительные примитивы для AI. Система загружается с Live ISO через протокол Limine и работает в Long Mode (x86_64).

> **Именование:** Название проекта **ospab** всегда пишется строчными буквами. Ядро — **AETERNA**.

---

## Системные компоненты

### Ядро: AETERNA
Микроядро отвечает за инициализацию оборудования (GDT, IDT, PIC, SSE/FPU), управление физической/виртуальной памятью, планировщик Compute-First и диспетчеризацию системных вызовов через MSR. Загрузка через протокол Limine, вывод через UEFI-фреймбуфер (шрифт VGA 8×16).

### Виртуальная файловая система (VFS + RamFS)
Полноценная POSIX-совместимая виртуальная файловая система. RamFS — файловая система в оперативной памяти, смонтированная в `/` со стандартными каталогами (`/etc`, `/proc`, `/dev`, `/home`, `/tmp`, `/sys`, `/boot`, `/var`). Все файловые операции (read, write, mkdir, touch, rm) реальные — никаких заглушек.

### Сетевой стек
- **RTL8139** — PCI-драйвер сетевой карты с DMA ring buffer TX/RX
- **Ethernet** — обработка кадров
- **ARP** — запрос/ответ с разрешением MAC-адресов
- **IPv4** — отправка/получение с контрольной суммой
- **ICMP** — ping (echo request/reply)
- **UDP** + **SNTP** — синхронизация времени

### Драйверы хранилища
- **ATA PIO** — доступ к IDE-дискам
- **AHCI SATA** — современные SATA-контроллеры через PCI BAR5

## Инструменты пользовательского пространства

Все утилиты написаны на Rust и пока работают как модули ядра (изоляция процессов — в планах).

### grape — Текстовый редактор
Полноэкранный текстовый редактор в стиле **nano** с интеграцией VFS.

- Полноэкранное редактирование с заголовком, строкой статуса и панелью горячих клавиш
- Горячие клавиши: `Ctrl+X` выход, `Ctrl+O` сохранить, `Ctrl+K` вырезать, `Ctrl+U` вставить, `Ctrl+W` поиск, `Ctrl+G` справка, `Ctrl+T` перейти к строке
- Стрелки, Home/End, PgUp/PgDn — навигация
- Горизонтальная/вертикальная прокрутка для больших файлов
- Поиск с оборотом
- Блочный курсор (инвертированные цвета)
- Автоматическое создание каталогов при сохранении
- Открытие файлов из VFS, сохранение в VFS

### tomato — Пакетный менеджер
Пакетный менеджер в стиле **pacman** с поддержкой локальных бинарных пакетов.

- `-S <pkg>` — Установить пакет (с разрешением зависимостей)
- `-R <pkg>` — Удалить пакет (очистить файлы)
- `-Q` — Список установленных пакетов
- `-Qi <pkg>` — Подробная информация о пакете
- `-Ss <запрос>` — Поиск доступных пакетов (без учёта регистра)
- `-Sy` — Синхронизация базы данных
- `-Syu` — Полное обновление системы
- Встроенный репозиторий с 10 пакетами: `base`, `coreutils`, `grape`, `plum`, `net-tools`, `tutor`, `kernel-headers`, `ospab-libc`, `man-pages`, `seed`
- Учёт установленных пакетов в `/var/lib/tomato/local/`
- Создание файлов в VFS при установке

### plum — Командная оболочка
POSIX-совместимая оболочка с **поддержкой bash скриптов** (v2.1.0+):

- **Переменные окружения**: `$VAR`, `${VAR}`, `$?` (код возврата)
- **`export VAR=value`** — установка/просмотр переменных окружения
- **`alias name=command`** — псевдонимы команд (`ll`, `la`, `cls`, `edit` и др.)
- **`unalias`**, **`unset`**, **`set`**, **`env`** — управление переменными
- **`type <cmd>`** — показать тип команды (встроенная, псевдоним, бинарный файл)
- **`source <file>`** или **`. <file>`** — выполнение скриптов из VFS
- **`bash <script.sh>`** — запуск bash скриптов
- Подстановка переменных во всех командах
- Раскрытие псевдонимов перед выполнением
- Цепочки команд через `;`
- Конфигурация при запуске: `/etc/plum/plumrc`
- Окружение по умолчанию: `HOME`, `USER`, `SHELL`, `PATH`, `PS1`, `TERM`, `EDITOR` и др.

**Возможности bash скриптов** (v2.1.0+):
- Условия: `if`/`then`/`else`/`fi`
- Циклы: `for`/`do`/`done`, `while`/`do`/`done`
- Функции: `function_name() { ... }`
- Подстановка переменных: `$var`, `${var}`
- Подстановка команд: `$(...)`
- Арифметическое выражение: `$((expr))`
- Цепочки команд через `;`
- Конфигурация при запуске: `/etc/plum/plumrc`
- Окружение по умолчанию: `HOME`, `USER`, `SHELL`, `PATH`, `PS1`, `TERM`, `EDITOR` и др.

### seed — Система инициализации
Первый логический процесс (аналог PID 1):

- **Регистрация сервисов** — 9 системных сервисов (kernel, vfs, scheduler, console, serial, keyboard, network, storage, plum)
- **`seed status`** — отображение всех сервисов с состоянием, PID, счётчиком перезапусков, описанием
- **`seed start/stop/restart <svc>`** — управление отдельными сервисами
- **`seed enable/disable <svc>`** — изменение политики активации
- **`seed log`** — показать журнал загрузки с временными метками
- Политики перезапуска: `always`, `once`, `manual`
- Файл конфигурации: `/etc/seed/init.conf`
- Запись конфигурации в VFS при инициализации

### tutor — Интерактивный туториал
Встроенный интерактивный гид по системе с темами: `intro`, `fs`, `net`, `mem`, `kernel`, `commands`.

## Структура проекта

```
ospab.os-v2/
├── arch/x86_64/        # HAL: GDT, IDT, PIC, SSE, клавиатура, фреймбуфер, serial
├── core/               # Планировщик, IPC, диспетчер системных вызовов
├── drivers/            # PCI, ATA, AHCI, заготовки VirtIO
├── executive/          # Менеджер объектов, процессы, управление питанием
├── hpc/                # Тензорный блок, DMA-движок, разделяемая память
├── mm/                 # Физический аллокатор, куча, VMM (4-уровневые таблицы страниц)
├── src/
│   ├── main.rs         # Точка входа — 5-фазная инициализация
│   ├── lib.rs          # Корень крейта — объявления модулей
│   ├── terminal.rs     # Интерактивный терминал с 28+ командами
│   ├── fs/             # VFS + RamFS (27+ узлов при загрузке)
│   ├── net/            # RTL8139, Ethernet, ARP, IPv4, ICMP, UDP, SNTP
│   └── drivers/        # ATA PIO, AHCI SATA
├── userland/
│   ├── grape/src/      # Текстовый редактор в стиле nano
│   ├── tomato/src/     # Пакетный менеджер в стиле pacman
│   ├── plum/src/       # POSIX-совместимая оболочка
│   └── seed/src/       # Система инициализации (PID 1)
├── limine.conf         # Конфигурация загрузчика
├── linker.ld           # Скрипт компоновщика ядра
├── x86_64-ospab.json   # Спецификация целевой платформы Rust
└── build.sh            # Скрипт сборки (cargo + xorriso)
```

## Последовательность загрузки

```
Фаза 0: Инициализация железа     — SSE, GDT, IDT, PIC
Фаза 1: Протокол Limine          — верификация загрузчика
Фаза 2: Память                   — физический аллокатор, куча 128 MiB, VMM
Фаза 3: Сервисы ядра             — планировщик, syscall MSR, VFS+RamFS, хранилище
Фаза 4: Сеть                     — автодетект RTL8139, ARP, ICMP-самотест
Фаза 4.5: Юзерленд               — seed (сервисы), plum (окружение + псевдонимы)
Фаза 5: Терминал                  — интерактивная консоль
```

## Сборка

```bash
# Требуется: Rust nightly, xorriso, mtools, limine
bash build.sh
```

## Запуск
```bash
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-19.iso \
  -m 256M \
  -serial stdio \
  -device rtl8139,netdev=net0 \
  -netdev user,id=net0
```
