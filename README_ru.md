**AETERNA** — это микроядерная операционная система, написанная полностью на Rust (`#![no_std]`).
Целевая архитектура — x86-64. Система загружается по протоколу [Limine](https://github.com/limine-bootloader/limine)
с гибридного BIOS/UEFI ISO и предоставляет полноценное пользовательское окружение —
оболочку, редактор, пакетный менеджер, систему инициализации и порт DOOM,
— всё работает на нулевом кольце без хостовой операционной системы.

Это не учебный проект. AETERNA реализует полноценную иерархию памяти (физический аллокатор,
128 МиБ кучи ядра, 4-уровневые таблицы страниц), POSIX-совместимую виртуальную файловую систему
с персистентностью на диск, рабочий сетевой стек, драйверы хранилища ATA/AHCI/NVMe,
аудиодрайверы (AC97, ES1371), UEFI-установщик на диск с GPT/FAT32/LFN и модель безопасности
на основе capability (в разработке).

> Название проекта **ospab** пишется строчными буквами. Ядро — **AETERNA**.

## Инициализация ядра

Ядро инициализируется в пяти последовательных фазах. Каждая фаза логируется
на серийный порт (COM1, 115200 бод) и в UEFI-фреймбуфер:

| Фаза | Название | Что происходит |
|---|---|---|
| 0 | Железо | SSE/FPU (CR4 `0x660`), GDT с TSS, IDT 256 обработчиков, PIC на IRQ 32–47 |
| 1 | Boot-протокол | Валидация ответов Limine, дескриптор фреймбуфера, карта памяти |
| 2 | Память | Физический аллокатор (bitmap), куча 128 МиБ, обход PML4 |
| 3 | Сервисы ядра | Планировщик, MSR-диспетчер сисколов, VFS + RamFS (27 узлов), определение накопителей |
| 4 | Сеть | PCI-поиск RTL8139/e1000, ARP-таблица, ICMP-самотест, UDP, SNTP |
| 4.5 | Юзерленд | seed (9 сервисов), plum (переменные окружения, псевдонимы) |
| 5 | Терминал | Интерактивная консоль — диспетчер команд, история, автодополнение |

Все фазы должны завершиться с `[OK]` до появления приглашения оболочки.
При ошибке ядро выводит структурированное сообщение на оба выхода и останавливается.

## Подсистемы

### Виртуальная файловая система

Полностью POSIX-совместимый слой VFS. `RamFS` — файловая система в памяти
на основе `BTreeMap`, защищённая `spin::Mutex`. При загрузке создаются стандартные каталоги:
`/etc`, `/proc`, `/dev`, `/home`, `/tmp`, `/sys`, `/boot`, `/var`.

Все файловые операции (`open`, `read`, `write`, `mkdir`, `unlink`, `rename`, `readdir`) —
полноценные реализации. Механизм отложенной синхронизации сериализует образ RamFS
на LBA 2048 загрузочного диска каждые 10 секунд (182 тика PIT),
обеспечивая сохранение игр DOOM и состояния оболочки между перезагрузками
без блокировки пользователя.

### Сетевой стек

| Слой | Реализация |
|---|---|
| Драйверы | RTL8139 (DMA ring TX/RX), Intel e1000 (PRO/1000 Gigabit) |
| Ethernet | Разбор кадров, таблица диспетчеризации по EtherType |
| ARP | Запрос/ответ с кэшем разрешения по TTL |
| IPv4 | Передача/приём с валидацией контрольной суммы заголовка |
| ICMP | Echo request/reply — `ping -c N <ip>` с RTT в микросекундах (TSC) |
| UDP | Отправка/приём датаграмм |
| SNTP | Однократная синхронизация времени (pool.ntp.org, UDP/123) |

### Хранилище

| Драйвер | Интерфейс | Примечания |
|---|---|---|
| ATA PIO | IDE / ISA | Опрос, 28-bit LBA, чтение/запись/identify |
| AHCI SATA | PCI BAR5 | NCQ, DMA, flush cache, сброс HBA |
| NVMe | PCI BAR0 | Очереди admin + I/O, PRP-передачи, identify namespace |

UEFI-установщик на диск записывает корректную таблицу разделов GPT, создаёт FAT32 ESP
с LFN-записями для `limine.conf`, копирует `BOOTX64.EFI`, образ ядра и конфигурацию
загрузчика, затем верифицирует каждый записанный сектор.

### Аудио

| Драйвер | Оборудование | Примечания |
|---|---|---|
| AC97 | Intel ICH (VMware, QEMU) | DMA BDL ring, 44100 Гц VRA, команда `soundtest` |
| ES1371 | Ensoniq AudioPCI / Creative CT5880 | Нативное устройство VMware Workstation |

## Пользовательское окружение

Все инструменты написаны на Rust и в настоящее время работают в режиме ядра.
Изоляция процессов (загрузчик ELF, Ring 3, ABI сисколов) — основная цель следующей фазы разработки.

### plum — Оболочка

POSIX-совместимая оболочка с полноценной поддержкой bash-скриптов.

- Переменные окружения: `$VAR`, `${VAR}`, `$?` (код возврата)
- `export`, `alias`/`unalias`, `unset`, `set`, `env`, `type`
- `source <файл>` / `. <файл>` — выполнение скриптов из VFS
- Условия: `if` / `then` / `else` / `fi`
- Циклы: `for` / `while` / `do` / `done`
- Функции, арифметическое расширение `$(( ))`, подстановка команд `$( )`
- Стартовая конфигурация в `/etc/plum/plumrc`

### grape — Текстовый редактор

Полноэкранный редактор в духе GNU nano с интеграцией VFS.

- Строка заголовка, статусная строка, строка подсказок
- `Ctrl+O` сохранить, `Ctrl+X` выход, `Ctrl+K` вырезать, `Ctrl+U` вставить, `Ctrl+W` поиск, `Ctrl+G` справка, `Ctrl+T` перейти к строке
- Стрелки, Home/End, PgUp/PgDn, горизонтальная и вертикальная прокрутка
- Поиск с оборотом, блочный курсор (инвертированные цвета)
- Автоматическое создание родительских каталогов при сохранении

### tomato — Пакетный менеджер

Менеджер пакетов в стиле pacman с поддержкой локальных бинарных пакетов.

| Флаг | Действие |
|---|---|
| `-S <pkg>` | Установить с разрешением зависимостей |
| `-R <pkg>` | Удалить и очистить файлы |
| `-Q` | Список установленных пакетов |
| `-Qi <pkg>` | Подробные метаданные пакета |
| `-Ss <запрос>` | Поиск доступных пакетов (без учёта регистра) |
| `-Sy` | Синхронизировать базу данных |
| `-Syu` | Полное обновление системы |

Встроенный репозиторий: `base` · `coreutils` · `grape` · `plum` · `net-tools` · `tutor` · `kernel-headers` · `ospab-libc` · `man-pages` · `seed`

### seed — Система инициализации

Логический PID 1, управляющий девятью системными сервисами.

| Команда | Описание |
|---|---|
| `seed status` | Все сервисы — состояние, PID, счётчик перезапусков |
| `seed start/stop/restart <svc>` | Управление отдельным сервисом |
| `seed enable/disable <svc>` | Изменить политику активации |
| `seed log` | Журнал загрузки с временными метками |

Сервисы: `kernel` · `vfs` · `scheduler` · `console` · `serial` · `keyboard` · `network` · `storage` · `plum`

### tutor — Интерактивный гид

Встроенный гид по системе. Темы: `intro` · `fs` · `net` · `mem` · `kernel` · `commands`

## DOOM

Порт [doomgeneric](https://github.com/ozkl/doomgeneric) — оригинальный движок DOOM 1993 года
работает напрямую на UEFI-фреймбуфере без хостовой ОС.

- Рендеринг 640×400 в нативном формате пикселей фреймбуфера (32-bit BGRA)
- Ввод с PS/2-клавиатуры с трансляцией скан-кодов
- Навигация по меню клавишами F1–F10
- Сохранение / загрузка через VFS (`/doom/doomsavN.dsg`), персистентность через отложенную синхронизацию
- Автономная Си-среда выполнения (`malloc`, `printf`, `memcpy`, `qsort`, …)
  целиком мостена к куче ядра Rust через FFI с `#[no_mangle]`

Запустить: `doom`

## Сборка

**Требования:** Rust nightly (`rustup default nightly`), `xorriso`, `mtools`, LLVM toolchain

```bash
git clone https://github.com/ospab/aeterna.git
cd aeterna
bash build.sh
```

Скрипт компилирует C-движок DOOM с `gcc -nostdlib`, собирает ядро Rust для `x86_64-ospab`,
снимает отладочные символы через `llvm-objcopy` и создаёт гибридный ISO под `isos/`.

Только ядро (без ISO):

```bash
bash build.sh kernel
```

## Запуск

```bash
# QEMU — BIOS-загрузка, с сетью и выводом на serial
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-103.iso \
  -m 256M \
  -serial stdio \
  -device rtl8139,netdev=net0 \
  -netdev user,id=net0

# QEMU — UEFI-загрузка
qemu-system-x86_64 \
  -cdrom isos/ospab-os-v2-103.iso \
  -m 256M \
  -serial stdio \
  -bios /usr/share/OVMF/OVMF_CODE.fd \
  -device rtl8139,netdev=net0 \
  -netdev user,id=net0

# VMware Workstation — подключить ISO как CD/DVD, прошивка UEFI
```

Вывод на stdout — полный структурированный журнал загрузки и все диагностические сообщения ядра.
