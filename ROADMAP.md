# План разработки ospab.os v2 (AETERNA) по tech-manifest

Ориентир — [tech-manifest.md](tech-manifest.md). Порядок приоритетов для MVP и первого CUI.

---

## Фаза 0: База (сделано)

- [x] Загрузчик Limine, протокол в ядре
- [x] x86_64 Long Mode, SSE/FPU (CR4 0x660)
- [x] GDT, TSS (double-fault stack)
- [x] Serial (COM1), ранние логи
- [x] Сборка + ISO (build.sh, hybrid BIOS+UEFI)
- [x] Логотип AETERNA (п.20 манифеста)

---

## Фаза 1: CUI и загрузка «как Linux» (текущая)

- [x] Логи при загрузке до появления терминала (п.20 — «логи как в Arch»)
- [x] Первый терминал (окружение поверх ядра) — приглашение, цикл
- [ ] Консоль: полный экран 80x25, перевод строки, скролл
- [ ] (Опционально) Клавиатура (polling/IRQ) и ввод в терминале

## Фаза 2: Ядро и память

- [ ] IDT, обработчики исключений и прерываний (п.5 — донорство из v1)

- [ ] Планировщик (заглушка/очередь) — п.10 Compute-First позже
- [ ] Модуль core::syscall (интерфейс, без полноценного диспетчера)

## Фаза 3: Микроядро и IPC

- [ ] Capabilities (п.7): заглушка/типы, без полноценной проверки
